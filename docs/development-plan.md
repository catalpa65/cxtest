# 电商项目开发计划（JDK 17 LTS）

## 目标

在稳定技术栈下完成一个可运行、可扩展、可演示的电商后端 MVP，并逐步补齐工程化能力。

## 技术选型

- 语言与框架：JDK 17 + Spring Boot 3.4.x
- 数据库：MySQL 8.x
- 缓存：Redis 7.x
- 持久层：MyBatis
- 消息队列：RabbitMQ（第 6 阶段引入）
- 部署：Docker Compose（后续阶段补齐）

## 里程碑（8 周）

1. 第 1 周：项目骨架 + 商品模块
- 完成统一返回模型、全局异常处理、参数校验
- 实现商品查询/创建/上下架 API
- 验收：Postman 可完整调用商品模块

2. 第 2 周：用户与鉴权
- 用户注册/登录
- 基于 JWT 的简单鉴权
- 验收：受保护接口需 token 才可访问

3. 第 3 周：购物车
- 加入购物车、修改数量、删除、查询
- Redis 优先（可降级到 DB）
- 验收：购物车状态与价格校验正确

4. 第 4-5 周：订单与库存
- 下单、取消、超时关单
- 库存预扣与回补
- 验收：并发下单库存不超卖（至少单机锁/乐观锁）

5. 第 6 周：支付与状态机
- 模拟支付回调
- 订单状态流转（待支付/已支付/已取消/已完成）
- 验收：重复回调幂等

6. 第 7 周：异步与可靠性
- RabbitMQ 处理延迟关单、异步通知
- 失败重试、死信队列
- 验收：关键异步链路可追踪可重试

7. 第 8 周：工程化收尾
- 集成测试、基础压测、日志与监控
- Docker Compose 一键启动
- 验收：README 中可复现完整流程

## 开发顺序（严格按步）

1. 基础框架（本次已开始）
2. 商品
3. 用户
4. 购物车
5. 订单
6. 支付
7. MQ 异步化
8. 监控与部署
